## 🚧 詰まった点・課題
- 動きの多いゲーム（例：ブロック崩し）では、処理が増えるにつれて挙動が不安定になることが多かった
- プログラミングの「部品」は理解できても、全体の組み立て方が曖昧だと感じた
- CSSが適用されない原因がファイル名の記述ミスだったなど、基本的なミスで時間を使う場面があった
- 基本情報技術者試験について、従来の勉強法では成果が出にくいと感じた

---

## 🔁 改善のために工夫したこと
- 生成AIへの指示を「#条件」「#形式」「#オプション」などに分解し、構造化したプロンプトを使用
- ゲーム制作では、画像や装飾を省き、数字やルールに集中することで実装難易度を下げた
- 基本情報技術者試験の勉強法を変更  
  - 科目Aを5問ずつ区切り、暗記 → 正解確認を繰り返す方式に切り替え
  - 計算問題は計算方法ではなく答えを覚える割り切った対策を採用
- 部品の組み立てを学ぶため、Scratchなど別アプローチの検討を開始
- 学習の優先順位を整理  
  （ポテパンキャンプ → 基本情報 → ドットインストール）

## 💡 学び・気づき
- HTML / CSS / JavaScript だけでも、工夫次第で多様なアプリやゲームを制作できる
- 生成AIは、遊び方やルールを最初に丁寧に説明すれば、オリジナルルールでも正確に実装できる
- プロンプトを細かく構造化することで、AIの出力精度が大きく向上する
- 学習は「量」だけでなく、「方法を変える勇気」が重要
- 学習や制作の発信を継続することで、第三者から評価される機会が生まれると実感した
